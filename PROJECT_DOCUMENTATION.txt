# 📘 Project Design & Architecture (PDA)
## Sharda University Student Platform — Helpdesk & Beyond

---

## 1. PROJECT OVERVIEW

**Project Name:** Sharda University Student Platform  
**Type:** Full-Stack Web Application  
**Stack:** MERN (MongoDB, Express.js, React.js, Node.js)  
**Purpose:** A centralized student support and engagement platform for Sharda University that combines a helpdesk ticketing system with events, opportunities, announcements, and AI-powered assistance.

---

## 2. OBJECTIVES

- Provide students with a single platform for all university support needs
- Streamline ticket-based support with role-based access
- Keep students informed through announcements and events
- Expose students to internships, hackathons, and opportunities
- Reduce repetitive queries using an AI/FAQ chatbot
- Enable password recovery and email notifications via Resend

---

## 6. DATABASE MODELS

### User
```
name, email, password (hashed), role (student/agent/admin/clubhead),
department, enrollmentId, avatar, isActive, lastLogin
```

### Ticket
```
ticketId (auto: SU-2024-0001), title, description, category, priority,
status, user (ref), assignedTo (ref), replies (sub-doc array),
attachments, tags, aiSuggestedPriority, resolvedAt, closedAt
```

### Reply (embedded in Ticket)
```
message, author (ref), authorRole, attachments, isInternal, createdAt
```

### Announcement
```
title, message, type (info/warning/urgent/success),
postedBy (ref), isActive, expiresAt
```

### Opportunity
```
title, description, type (Internship/Hackathon/Competition/Workshop/Job/Scholarship),
company, location, externalLink, deadline, stipend, eligibility,
tags, postedBy (ref), bookmarks (user refs array), isActive
```

### Event
```
title, description, category, date, endDate, venue, poster,
videoLink, registrationLink, organizer, maxParticipants,
tags, postedBy (ref), interestedUsers (user refs array), isActive
```

### PasswordReset
```
user (ref), token, expiresAt (TTL: 15min), used
```

## 9. KEY FEATURES

### 🎫 Helpdesk Ticketing
- Auto-generated ticket IDs (SU-2024-0001)
- 8 categories: IT Support, Administration, Hostel, Library, Finance, Academic, Infrastructure, Other
- 4 priorities: Low, Medium, High, Critical
- 4 statuses: Open → In Progress → Resolved → Closed
- Chat-style threaded replies
- Internal notes (staff only)
- File attachments (up to 5MB)
- AI-suggested priority based on keywords

### 📢 Announcements
- Types: Info, Warning, Urgent, Success
- Color-coded banners on dashboard
- Admin/Agent can post and delete

### 💼 Opportunities
- 6 types: Internship, Hackathon, Competition, Workshop, Job, Scholarship
- Deadline countdown badges (🔥 urgent, ⚡ soon)
- Bookmark/save for later
- Search and filter by type

### 🎉 Events
- 7 categories with emoji icons
- Poster image upload
- Interest toggle with count
- Live status: Today!, Tomorrow, In Nd, Ongoing, Completed
- Registration and video links
- Upcoming/Past/All filter tabs

### 🤖 AI Assistant
- Chat-based interface
- FAQ keyword matching
- Suggested questions
- Auto-detects ticket creation intent
- Graceful fallback with ticket CTA

### 🔐 Auth & Security
- JWT authentication (7-day expiry)
- bcrypt password hashing (10 salt rounds)
- Role-based route protection
- Forgot password via Resend email
- Reset token auto-expires in 15 minutes

## 11. SETUP INSTRUCTIONS

### Prerequisites
- Node.js v18+
- MongoDB (local or Atlas)
- npm v9+

### Backend Setup
```bash
cd backend
npm install
cp .env.example .env
# Edit .env with your values
node utils/seed.js        # Seed demo data
npm run dev               # Start server
```

### Frontend Setup
```bash
cd frontend
npm install
cp .env.example .env
# Set REACT_APP_API_URL=http://localhost:8080/api
npm start
```
